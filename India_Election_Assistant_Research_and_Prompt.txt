# PART 1: RESEARCH FINDINGS

## 1.1 The Indian Election Landscape

India is the world's largest democracy, with over **968 million eligible voters** as of the 2024 General Elections. The Election Commission of India (ECI) is the apex constitutional authority governing elections. Understanding this ecosystem is foundational to building an effective education assistant.

### Types of Elections in India

**1. Lok Sabha Elections (General Elections)**
- Held every 5 years to elect 543 Members of Parliament (MPs)
- Uses First-Past-The-Post (FPTP) system
- Citizens aged 18+ vote directly for constituency representatives
- The party/alliance with majority forms the central government

**2. Rajya Sabha Elections (Upper House)**
- Members elected by State Legislative Assembly (MLA) members — NOT directly by citizens
- Election every 2 years for one-third of seats
- Members serve 6-year terms

**3. State Legislative Assembly (Vidhan Sabha) Elections**
- Every 5 years per state (different states have different dates)
- Citizens elect MLAs for their state government
- Party/alliance with majority forms the state government

**4. Panchayat Elections**
- Local self-government at village level
- Every 5 years
- Conducted by State Election Commissions

**5. Municipal Corporation Elections**
- Urban local government
- Every 5 years
- Conducted by State Election Commissions

**6. By-Elections (Bypolls)**
- Held when a seat falls vacant due to death, resignation, or disqualification
- Only the specific constituency votes

## 1.2 The Voter Registration Process (Detailed)

### Eligibility Criteria
- Indian citizen
- Minimum age: **18 years** on the qualifying date (1st January, 1st April, 1st July, or 1st October of the revision year)
- Ordinary resident of the polling area/constituency

### Registration Forms
| Form | Purpose |
|------|---------|
| Form 6 | New voter registration (general voters, first-timers, constituency transfers) |
| Form 6A | NRI voter registration |
| Form 7 | Deletion or objection in electoral roll |
| Form 8 | Change of address / correction of entries / EPIC replacement / PwD marking |

### Online Registration Steps
1. Visit **https://voters.eci.gov.in/** or download the **Voter Helpline App**
2. Create an account with mobile number
3. Fill Form 6 with: name, DOB, address, ID documents
4. Upload supporting documents (address proof, age proof, photo)
5. Submit application — receive acknowledgement number
6. Booth Level Officer (BLO) may visit for verification
7. Name appears in electoral roll after approval

### Offline Registration Steps
1. Obtain Form 6 free from Electoral Registration Officer (ERO) / BLO offices
2. Fill two copies with supporting documents
3. Submit at ERO/AERO office in person or by post

### Important Documents Required
- Age proof: Birth certificate, school certificate, passport, Aadhaar
- Address proof: Aadhaar, utility bill, bank passbook, rent agreement
- Passport-size photograph

### Checking Registration Status
- Visit **https://electoralsearch.eci.gov.in/**
- Use Voter Helpline App (1950 toll-free number)
- Provide name, state, constituency, gender

### E-EPIC (Electronic Electoral Photo Identity Card)
- Digital version of Voter ID card downloadable as PDF
- Accessible from **voters.eci.gov.in**
- Can be used as identity proof at polling stations

## 1.3 The Voting Day Process (Step-by-Step)

### Before Voting Day
1. Check name in electoral roll
2. Know your polling station (within 2km of your address)
3. Download/carry Voter ID (EPIC) or approved alternative ID
4. Check voting date for your phase (Lok Sabha has multiple phases)

### Approved Alternative ID Documents (if EPIC not available)
- Aadhaar Card
- Passport
- Driving Licence
- PAN Card
- MNREGA Job Card
- Smart Card issued by RGI under NPR
- Pension document with photo
- Service identity cards with photo

### At the Polling Station
1. Join the correct queue (often separate queues for men/women/PwD)
2. Name verified against electoral roll
3. Identity document checked
4. Indelible ink applied to left index finger
5. Voter slip issued
6. Proceed to EVM (Electronic Voting Machine)
7. Presiding Officer activates the Ballot Unit
8. Press button next to chosen candidate's name and party symbol
9. Green light + beep confirms vote is cast
10. VVPAT (Voter Verifiable Paper Audit Trail) slip appears for 7 seconds for verification

### EVM (Electronic Voting Machine) Explained
- Consists of Control Unit (operated by officer) and Ballot Unit (used by voter)
- Ballistic unit shows candidate names with party symbols
- Voter presses button ? green light + beep = confirmation
- VVPAT prints a paper receipt visible for 7 seconds (non-retrievable)

### Special Provisions
- **Elderly (80+)**: Can vote via postal ballot or home visit by polling official
- **PwD (Persons with Disabilities)**: Wheelchair access, companion assistance allowed, Braille marking on EVMs
- **Absentee voters**: Certain categories can use postal ballot
- **Home voting**: New pilot provision being expanded for elderly and PwD

## 1.4 Election Timeline (Full Election Cycle)

```
ANNOUNCEMENT PHASE
+-- ECI holds press conference ? announces election schedule
+-- Model Code of Conduct (MCC) comes into effect immediately
+-- Schedule includes: notification dates, last date for filing nominations, scrutiny, withdrawal, polling date(s), counting date

NOMINATION PHASE (typically 4–7 days before polling)
+-- Candidates file nomination papers with Returning Officer
+-- Security deposit: ?10,000 (Lok Sabha), ?5,000 (Vidhan Sabha)
¦   (SC/ST candidates pay half)
+-- Proposers required: 1 for recognized party; 10 for independent/unrecognized party

SCRUTINY OF NOMINATIONS
+-- Returning Officer examines nomination papers for validity

WITHDRAWAL OF CANDIDATURE
+-- Last date to withdraw (typically 2 days after scrutiny)

CAMPAIGN PERIOD
+-- Political parties campaign, hold rallies
+-- Campaign expenditure monitored by ECI
+-- Campaign STOPS 48 hours before polling ("campaign silence period")

POLLING DAY
+-- Polling stations open for minimum 8 hours
+-- Each station handles max 1,200 voters
+-- EVM + VVPAT used for voting

COUNTING DAY
+-- EVMs brought to counting centers under security
+-- Postal ballots counted first
+-- EVM tallies counted round-by-round, constituency-by-constituency

RESULT DECLARATION
+-- Winning candidate declared by Returning Officer
+-- President/Governor convenes respective house

POST-ELECTION
+-- Election petitions may be filed in High Court within 45 days
```

## 1.5 Key Problems Voters Face (Research-Backed)

### For Ordinary Voters
1. **Registration confusion**: Not knowing which form to use, missing qualifying dates
2. **Errors in voter list**: Name misspelled, wrong address, or name missing entirely
3. **Lack of awareness**: Not knowing polling station location, date, or valid ID
4. **Language barriers**: ECI materials often in English/Hindi, not local languages
5. **Misinformation**: Fake voting websites and misleading information online

### For First-Time Voters (18+)
1. **Don't know where to register** or which portal is legitimate
2. **Documents confusion**: Not knowing what documents are required
3. **Qualifying date confusion**: Missing the window to register

### For Elderly Voters (60+, especially 80+)
1. **Physical mobility challenges** at polling stations
2. **Unfamiliarity with digital portals** for registration
3. **Not aware of home voting provisions**
4. **Difficulty reading small print** on EVMs

### For PwD (Persons with Disabilities)
1. **Lack of accessible polling booths**
2. **No Braille ballot availability** in many booths
3. **Limited wheelchair ramps**

### For Election Officials / Polling Staff
1. **Verification delays** — manual roll checking is slow
2. **Managing long queues** on voting day
3. **Lack of digital resources** to educate voters on-the-spot
4. **Handling complaints** about voter list errors

### For Migrant Workers
1. **Registered in native state** but residing elsewhere — cannot vote without travel
2. **No mechanism** to easily transfer constituency

### Systemic Problems
1. **Misinformation about EVMs**: Allegations of tampering undermine voter confidence
2. **Multiple voting websites** creating confusion about which is official
3. **Low voter turnout** among youth, urban areas
4. **Distrust in political system**: Feeling that votes don't matter

## 2.5 Core Agent Capabilities

### 1. Persona-Adaptive Communication
The agent detects the user type (elderly, youth, official, NRI) and adjusts:
- Vocabulary simplicity
- Response length
- Use of analogies
- Font size recommendations
- Language formality

### 2. Multilingual Support
- UI fully translates to selected language
- Agent responds in the user's selected language
- Voice input/output in regional languages via Web Speech API

### 3. Guided Journeys
Structured conversation flows for:
- "I want to register to vote" ? 8-step guided journey
- "I want to vote today" ? 5-step voting day guide
- "I have a problem with my voter ID" ? Diagnosis + resolution path

### 4. Visual Education
- Interactive EVM explainer
- Flowchart of election process
- Step-by-step registration with progress tracker
- Infographic cards for different election types

### 5. Accessibility Features
- Large text mode (for elderly)
- High contrast mode
- Read-aloud (TTS) for every agent response
- Voice input for mobility-impaired users
- Simple language mode ("explain like I'm 60")

### 6. Official Redirects
- Never misinform; always redirect to official ECI URLs
- Direct links to voter registration portal
- ECI helpline number (1950) prominently displayed
- cVIGIL app links for violation reporting

# PART 4: SPECIAL FEATURES (COMPETITION BOOSTERS)

## 4.1 Unique Features to Stand Out

### 1. "Explain Like I'm..." Mode
User selects their comfort level:
- "Explain like I'm 10" ? very simple language, use analogies
- "Explain like I'm 18" ? standard youth language
- "Explain like I'm 60" ? simple, large text, slow pacing
- "I'm an expert" ? detailed constitutional/legal info

### 2. Step-by-Step Registration Companion
A guided, interactive multi-step form that walks the user through the ECI registration process with real-time validation of what documents they'd need.

### 3. Myth Busting Module
Addresses common misconceptions:
- "My vote doesn't matter" ? Shows how close margins change outcomes
- "EVMs can be hacked" ? ECI's official explanations
- "I need money to vote" ? Voting is free and your right
- "If I miss this election, I lose my voter card" ? Correct misconception

### 4. Live Election Calendar
Displays upcoming election dates for the user's state (pulled from ECI data or hardcoded from ECI calendar) with reminders about registration deadlines.

### 5. Document Checklist Generator
User answers 3 questions (new/existing voter, has Aadhaar or not, age) ? System generates a personalized "What to bring" checklist.

### 6. Polling Booth Locator
Links directly to ECI's official polling station finder with guidance on how to use it.

### 7. cVIGIL Integration Education
Educates voters about ECI's official violation-reporting app (cVIGIL) and how to use it to report election code violations.

### 8. Voice-First Mode
Designed for low-literacy users and elderly — complete interaction via voice, with very simple language responses also spoken back aloud.

## ACCESSIBILITY REQUIREMENTS (Critical)

Implement ALL of the following:

1. **Text Size Control**: Three sizes — Normal / Large / Extra Large. Persists in localStorage.

2. **Read Aloud**: Every agent message has a small speaker icon. Clicking it reads the message aloud using Web Speech API in the user's selected language.

3. **Voice Input**: Microphone button in chat input. Uses SpeechRecognition API. Shows waveform animation while recording. Detected language matches user's selected language.

4. **High Contrast Mode**: Toggle in settings. Switches to pure black on white, removes gray tones, increases border widths.

5. **Reduced Motion Mode**: Respects `prefers-reduced-motion` CSS media query. Disables all animations for users who need it.

6. **Keyboard Navigation**: Full keyboard navigability. All interactive elements have visible focus states. Tab order is logical.

7. **Screen Reader Support**: All images have alt text. Chat messages have proper ARIA labels. Buttons have descriptive aria-label attributes.

8. **Simple Language Mode**: Toggle in settings. When enabled, the agent prompt instructs the LLM to use only simple vocabulary (Grade 5-6 reading level), avoid jargon, and use short sentences.

9. **Large Touch Targets**: All buttons minimum 44×44px. Special "elderly mode" increases this to 56×56px.

10. **Offline Basics**: Cache the core knowledge base (election process steps, form information, helpline numbers) in localStorage so basic questions can be answered without internet.

## LLM INTEGRATION

```javascript
// llmService.js

const GEMINI_API_URL = 'https://generativelanguage.googleapis.com/v1beta/models/gemini-1.5-flash:generateContent';
const GROQ_API_URL = 'https://api.groq.com/openai/v1/chat/completions';

async function callGemini(messages, systemPrompt) {
  const response = await fetch(`${GEMINI_API_URL}?key=${import.meta.env.VITE_GEMINI_API_KEY}`, {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({
      system_instruction: { parts: [{ text: systemPrompt }] },
      contents: messages.map(m => ({
        role: m.role === 'assistant' ? 'model' : 'user',
        parts: [{ text: m.content }]
      })),
      generationConfig: {
        temperature: 0.3, // Lower for factual accuracy
        maxOutputTokens: 1024,
      }
    })
  });
  const data = await response.json();
  return data.candidates[0].content.parts[0].text;
}

async function callGroq(messages, systemPrompt) {
  const response = await fetch(GROQ_API_URL, {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'Authorization': `Bearer ${import.meta.env.VITE_GROQ_API_KEY}`
    },
    body: JSON.stringify({
      model: 'llama3-8b-8192',
      messages: [
        { role: 'system', content: systemPrompt },
        ...messages
      ],
      temperature: 0.3,
      max_tokens: 1024
    })
  });
  const data = await response.json();
  return data.choices[0].message.content;
}

// Primary: Gemini, Fallback: Groq
export async function getAgentResponse(messages, systemPrompt) {
  try {
    return await callGemini(messages, systemPrompt);
  } catch (error) {
    console.error('Gemini failed, trying Groq:', error);
    return await callGroq(messages, systemPrompt);
  }
}
```
